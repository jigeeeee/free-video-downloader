# 竞品调研报告：BibiGPT & NoteGPT

> 调研日期：2026-06-27  
> 调研对象：[bibigpt.co](https://bibigpt.co/)、[notegpt.io/cn/bilibili-summarizer](https://notegpt.io/cn/bilibili-summarizer)  
> 项目背景：当前项目是「万能视频下载器」(yt-dlp 内核)，已支持 YouTube/Bilibili/Douyin 三大平台下载。Phase 4+ 规划了字幕、AI 总结、批量下载、格式转换等功能。  
> 调研目的：弄清 AI 视频工具赛道的真实产品形态、可落地的差异化机会，为后续功能扩展决策提供依据。

---

## 一、BibiGPT 详细分析

### 1.1 产品定位

- **品牌**：BibiGPT（最初叫 BiliGPT，也叫"AI 课代表"）
- **创始人**：JimmyLv（吕立青），2023 年创立
- **用户规模**：100 万+ 学习者和创作者，累计总结 500 万+ 视频
- **一句话价值主张**：「1 小时视频，3 分钟看完重点——AI 视频总结、思维导图、视频转文字，一键搞定」

### 1.2 核心交互流程（3 步）

```
粘贴链接 → AI 视频总结 → 知识随你用
```

| 步骤 | 关键动作 | 输出 |
|------|---------|------|
| 1. 粘贴链接 | 输入 30+ 平台链接或上传本地音视频 | 视频元数据可视化 |
| 2. AI 视频总结 | 自动提取字幕、生成带时间戳的总结、章节速览 | 结构化总结 |
| 3. 知识随你用 | 思维导图 / AI 追问 / 图文改写 / 一键保存到 Notion、Obsidian、flomo | 二次加工产出 |

### 1.3 支持的平台

- **视频类**：YouTube、Bilibili、TikTok、Douyin（抖音）、Kuaishou（快手）、Xiaohongshu（小红书）、Twitter/X、Vimeo、TED、Coursera、Khan Academy
- **音频类**：Apple Podcasts、Spotify、Xiaoyuzhou（小宇宙）、Ximalaya（喜马拉雅）
- **本地文件**：mp3、mp4、m4a、wav、srt、vtt、ass
- **任意网页**：带音视频标签的网页

### 1.4 核心能力（Capabilities）

- 带时间戳的视频总结
- 完整音频转录提取
- 思维导图生成
- 基于视频内容的多轮 AI 聊天（"AI 追问"）
- 文章改写（公众号 / 小红书 / Twitter / blog 多平台风格）
- 字幕翻译与烧录（视频内嵌字幕）
- 视觉内容分析（PPT 幻灯片 / 帧理解）
- Notion / Obsidian / 飞书同步
- MCP server（让 AI Agent 调用）
- OpenAPI REST API
- NLWeb `/ask` 端点（SSE 流式问答）
- Anthropic Skill（`npx skills add JimmyLv/bibigpt-skill`）

### 1.5 客户端形态（全平台覆盖）

| 端 | 形态 |
|----|------|
| Web | bibigpt.co（中）/ aitodo.co（国际） |
| 浏览器 | Chrome / Firefox / Edge 扩展 |
| 桌面 | macOS / Windows（Tauri 2.x） |
| 移动 | iOS / Android（Expo） |

### 1.6 商业模式

| 套餐 | 价格 | 权益 |
|------|------|------|
| 免费 | ¥0 | 每日配额（最长 30 分钟视频） |
| Plus | $19.8/月（年付 $142.56） | 高配额 |
| Pro | $34.8/月（年付 $250.56） | 含视觉分析、批处理、API 访问 |
| Lifetime | $888（限量） | 终身买断 |
| Pay-as-you-go | $20-$200 | 2000-40000 credits，量大 50% off |

### 1.7 数据策略（合规亮点）

- 上传文件 30 天后自动删除
- 总结默认私有，仅在用户明确切换时公开
- 不在未经同意时训练用户内容
- 全量数据导出 + 删除功能
- GDPR / CCPA 合规

## 二、NoteGPT 详细分析

### 2.1 产品定位

- **品牌**：NoteGPT
- **入口**：`notegpt.io/cn/bilibili-summarizer`（主推 B 站视频总结）
- **页面标题**：「BiliBili 视频总结工具 - 一键总结哔哩哔哩视频」
- **核心心智**：「3 分钟看完 1 小时视频」「免费开始」

### 2.2 核心交互流程

```
粘贴 B 站链接 → AI 一键总结 → 输出文字总结 + 字幕文本
```

- 主打 B 站，但页面同时支持 YouTube、抖音等其他平台
- 落地页核心是「粘贴链接 → 一键总结」的极简两步
- 输出形态：AI 文字总结 + 章节大纲 + 时间戳

### 2.3 产品形态

- 主要是 Web 端，没有看到明显的浏览器扩展 / 桌面端布局
- UI 比 BibiGPT 更轻量、更聚焦"B 站学习场景"
- 强调「免费」「无需下载安装」

## 三、两家竞品对比

### 3.1 共同点

| 维度 | 共识 |
|------|------|
| 核心心智 | "粘贴链接 → AI 一键总结"，把长视频压成短结论 |
| AI 总结形态 | 时间戳 + 章节大纲 + 思维导图，几乎是标配 |
| 多平台 | 至少覆盖 YouTube / Bilibili / 国内短视频 |
| 免费 + 付费 | 都用 free tier 获客，再引导订阅 |
| AI 追问 | 基于视频内容的多轮对话是核心差异化 |
| 知识沉淀 | 都做笔记导出 / 保存到外部工具 |
| 总结可编辑 | 总结结果允许二次加工（改写 / 导出） |

### 3.2 差异点

| 维度 | BibiGPT | NoteGPT |
|------|---------|---------|
| 平台广度 | 30+ 平台 + 本地文件 + 任意网页 | 主打 B 站，覆盖 10+ 平台 |
| 国际化 | 双站点（bibigpt.co + aitodo.co）| 中国市场为主 |
| 客户端形态 | Web + 扩展 + 桌面 + 移动，全平台 | 主要是 Web |
| AI Agent 化 | MCP / OpenAPI / NLWeb / Anthropic Skill（最前沿）| 较基础 |
| 商业模式 | 订阅 + 终身 + 积分（更激进）| 免费试用 + 订阅 |
| 笔记集成 | Notion / Obsidian / 飞书 / flomo | 较基础 |
| 视觉分析 | 有（PPT / 帧理解）| 较简单 |
| 数据合规 | 30 天自动删除 + GDPR 突出 | 不突出 |
| 品牌定位 | "AI 音视频知行助理"（效率工具）| "AI 笔记工具"（学习工具）|
| 内容改写 | 多平台风格改写 | 较基础 |

## 四、我们项目的差异化机会分析

### 4.1 我们当前的"基因"和竞品的差异

| 维度 | 我们 | BibiGPT | NoteGPT |
|------|------|---------|---------|
| 内核能力 | yt-dlp 真实下载到本地 | 不下载，只总结 | 不下载，只总结 |
| 文件形态 | 拿到 .mp4 / .mp3 文件 | 没有文件，只有总结 | 同左 |
| 核心心智 | "工具型"：拿到文件 | "内容型"：看懂内容 | 同左 |
| 用户场景 | 离线保存 / 二次剪辑 / 素材库 | 学习 / 复习 | 学习 / 复习 |

**核心洞察**：竞品都没有真正"把视频变成你的本地文件"。我们手里有文件，这是天然优势，但目前完全没利用。

### 4.2 三大产品演进方向（推荐路径）

#### 方向 A：下载 + AI 增值服务（主推）

把"下载"从终点变成起点，让用户在下载完之后立刻得到 AI 加工服务。

| 新功能 | 用户价值 | 实现思路 |
|--------|---------|----------|
| 视频转文字 | 拿到字幕稿 | yt-dlp 抽字幕 → DeepSeek 转写 |
| AI 总结 | 下载完立刻看重点 | DeepSeek API 总结 |
| 思维导图 | 可视化视频结构 | DeepSeek 输出 Mermaid / Markdown |
| AI 问答 | 看完再追问细节 | DeepSeek 多轮对话 |
| 内容改写 | 一键变公众号 / 小红书 / Twitter 文案 | DeepSeek 风格化改写 |

#### 方向 B：批量 + 队列（效率工具升级）

把单条下载升级到「提交一批、坐着等结果」。

| 新功能 | 用户价值 |
|--------|----------|
| 批量粘贴多链接 | 一次提交 N 个视频 |
| 队列展示 | 看哪些在跑、哪些排队、哪些完成 |
| 后台任务 | 关掉浏览器也能继续下载 |

#### 方向 C：多格式转换（实用工具扩展）

把下载的文件做更多转换。

| 新功能 | 用户价值 |
|--------|----------|
| 提取音频为 MP3 | 拿来做播客 / 听书 |
| 视频转 GIF | 截取高光片段 |
| 格式互转（MP4/MKV/WebM） | 兼容性 |

### 4.3 推荐的产品愿景（融合 A+B+C）

> **「万能视频下载器」= 一站式音视频生产力工具**
>
> 不只是下载，而是下载 + 理解 + 加工 + 二次创作

| 阶段 | 能力 | 比喻 |
|------|------|------|
| L1（已有）| 链接 → 本地文件 | 快递员 |
| L2（推荐）| 本地文件 → AI 总结 / 字幕 / 思维导图 | 快递员 + 翻译 |
| L3（推荐）| 多链接 → 队列 + 通知 | 快递员 + 调度中心 |
| L4（远期）| AI 改写 / 风格化输出 | 快递员 + 编辑 |

## 五、竞品给我们的具体行动建议

### 5.1 必须做（基础 AI 能力）

1. **AI 视频总结**：借鉴 BibiGPT 时间戳 + 章节大纲形态
2. **字幕提取与翻译**：yt-dlp 已经有字幕模块，需要暴露给前端
3. **思维导图**：Mermaid 渲染最简单，DeepSeek 输出 Mermaid 语法即可

### 5.2 应该做（差异化能力）

4. **本地文件 AI 处理**：用户上传本地视频也能总结（竞品需要粘链接，我们有文件）
5. **AI 改写**：针对公众号 / 小红书 / Twitter 不同平台风格
6. **批量队列**：BibiGPT 没有，这是我们的独门优势

### 5.3 值得做（增值 / 商业化）

7. **订阅 + 积分制**：参考 BibiGPT 的 4 档套餐
8. **MCP / OpenAPI 化**：让 Cursor / Claude Code 等 AI 工具调用我们
9. **数据合规**：自动删除 + GDPR，是出海必需

### 5.4 不建议做

- **浏览器扩展 / 桌面端**：开发成本高，BibiGPT 做了我们短期没必要
- **移动 App**：同理
- **多语言国际版**：除非有明确海外用户

## 七、结论

> **BibiGPT 是行业天花板**：能力广、客户端全、商业化深、Agent 化超前。我们短期内不应对标它做"全能"，但应学习它的**核心三件套（总结 / 思维导图 / AI 追问）**。
>
> **NoteGPT 是直接的竞品**：打法轻、B 站聚焦、Web 为主。我们要差异化就必须靠我们的**下载文件**这个独门优势，把 AI 服务嫁接到本地文件上。
>
> **我们的最佳策略**：
> 1. 先做"下载 + AI 增值"（方向 A），把视频文件变成"可总结、可问答、可改写"的知识资产
> 2. 同时做"批量 + 队列"（方向 B），效率工具的天然延伸
> 3. 商业化参考 BibiGPT 的积分制，比订阅制更适合工具型用户

## 八、当前落地状态（2026-06-29 更新）

根据本次开发进展，竞品调研中建议的部分能力已经落地：

| 能力 | 当前状态 |
|------|----------|
| 下载 + AI 增值 | 已接入字幕、总结、思维导图、问答、翻译、改写 |
| 批量 + 队列 | 已有批量提交、统一任务中心、SQLite 任务持久化 |
| 本地文件处理 | 已有上传转录、ffmpeg 转换、音频提取和压缩入口 |
| 内容改写 | 已支持学习笔记、公众号、小红书、Twitter/X、Markdown 风格 |
| 历史记录 | 文件历史和 AI 结果写入 SQLite，可通过 `/api/history` 查询 |

仍建议继续推进：

1. 无字幕视频自动 Whisper fallback。
2. AI 结果导出到 Markdown/TXT。
3. 批量队列页面增强。
4. MCP/OpenAPI 化，方便 AI Agent 调用。
5. 积分/额度模型，用于后续商业化预留。
